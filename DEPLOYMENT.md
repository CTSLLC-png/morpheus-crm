# Morpheus CRM — Sprint 1 Deployment Guide
**Certified Training Standards · Albany, NY · morpheuscr.com**

---

## What's in this package

| File / Folder | Purpose |
|---|---|
| `src/lib/supabase.js` | Supabase client + auth helpers |
| `src/lib/ai.js` | Anthropic API — scenario gen, caller roleplay, scoring |
| `src/lib/db.js` | All database reads/writes (participants, sessions, scores, certs) |
| `src/hooks/useAuth.jsx` | Global auth context (session, role, participantId) |
| `src/hooks/useCallSession.js` | Full call lifecycle hook — AI + DB wired together |
| `src/App.jsx` | Root router — auth-aware, role-aware routing |
| `src/pages/Login.jsx` | Login + password reset page |
| `supabase/functions/on-user-signup/` | Edge function — validates and stamps user role on signup |
| `morpheus_schema_v1.sql` | Full Supabase database schema (run first) |
| `vercel.json` | Vercel SPA routing + security headers |

## Step 1 — Set up Supabase

1. Go to **supabase.com** → New project → name it `morpheus-crm`
2. Open **SQL Editor** → paste `morpheus_schema_v1.sql` → **Run**
3. Confirm these tables exist:
   - `staff_profiles`, `participants`, `cohorts`, `cohort_enrollments`
   - `call_sessions`, `call_scores`, `score_matrix_weights`, `certifications`
4. Go to **Settings → API** → copy:
   - Project URL → `VITE_SUPABASE_URL`
   - `anon` public key → `VITE_SUPABASE_ANON_KEY`
5. Go to **Authentication → URL Configuration** → set:
   - Site URL: `https://morpheuscr.com`
   - Redirect URLs: `https://morpheuscr.com/reset-password`

---

## Step 2 — Deploy the Auth Edge Function

```bash
# Install Supabase CLI
npm install -g supabase

# Login
supabase login

# Link to your project
supabase link --project-ref YOUR_PROJECT_REF

# Deploy the signup hook
supabase functions deploy on-user-signup
```

Then in Supabase Dashboard:
**Authentication → Hooks → After user is created** → select `on-user-signup`

## Step 3 — Configure environment variables

```bash
# In the morpheus project folder:
cp .env.example .env.local
```

Edit `.env.local`:
```
VITE_SUPABASE_URL=https://YOUR_PROJECT_REF.supabase.co
VITE_SUPABASE_ANON_KEY=your_anon_key_here
VITE_ANTHROPIC_API_KEY=your_anthropic_key_here
VITE_APP_URL=https://morpheuscr.com
```

Get your Anthropic API key at: **console.anthropic.com → API Keys**

## Step 4 — Create your first staff accounts

In Supabase Dashboard → **Authentication → Users → Add user**:

| Email | Password | Role (in user_metadata) |
|---|---|---|
| `twilliams@morpheuscr.com` | (set strong password) | `trainer` |
| `admin@morpheuscr.com` | (set strong password) | `super_admin` |

Or using the Supabase client (run once from your local machine):
```js
import { createClient } from '@supabase/supabase-js'
const admin = createClient(URL, SERVICE_ROLE_KEY)

await admin.auth.admin.createUser({
  email: 'twilliams@morpheuscr.com',
  password: 'SecurePassword123!',
  user_metadata: { role: 'trainer' },
  email_confirm: true,
})
```

Then create the matching `staff_profiles` row:
```sql
INSERT INTO staff_profiles (user_id, full_name, title)
VALUES ('<user_id_from_above>', 'T. Williams', 'Lead Trainer');
```

## Step 5 — Create participant accounts

From the trainer admin panel (Sprint 2 UI) or via Supabase:
```js
await admin.auth.admin.createUser({
  email: 'jordan.m@example.com',
  password: 'TempPassword123!',
  user_metadata: { role: 'participant' },
  email_confirm: true,
})
```

Then insert the participant record:
```sql
INSERT INTO participants
  (user_id, full_name, dob, program_source, ldss_office, ldss_case_number, ldss_caseworker, assigned_trainer)
VALUES
  ('<auth_user_id>', 'Jordan Marcus', '1992-08-14',
   'LDSS Albany', 'LDSS Albany – Workforce Solutions',
   'ALB-WF-2025-1104', 'Ms. R. Henderson',
   '<staff_profile_id>');
```

The `cts_id` (e.g. `CTS-00001`) is auto-generated by the database trigger.

## Step 6 — Deploy to Vercel

```bash
# Install Vercel CLI
npm install -g vercel

# In the morpheus project folder:
npm install
vercel

# Follow prompts — Vercel will detect Vite automatically
# Framework: Vite
# Build command: npm run build
# Output directory: dist
```

Add environment variables in **Vercel Dashboard → Project → Settings → Environment Variables**:
- `VITE_SUPABASE_URL`
- `VITE_SUPABASE_ANON_KEY`
- `VITE_ANTHROPIC_API_KEY`
- `VITE_APP_URL`

**Custom domain:**
Vercel Dashboard → Domains → Add `morpheuscr.com`
Point your domain registrar's nameservers to Vercel, or add the CNAME record provided.

## Step 7 — Verify the full loop

1. Log in as a **trainer** at `morpheuscr.com`
2. Navigate to **AI Call Simulator**
3. Select a participant, generate a scenario, complete a call
4. Confirm scores appear and "Saved to Morpheus" indicator fires
5. Log out → log in as a **participant**
6. Confirm their call history shows the session just completed
7. Run a self-serve call as the participant
8. Confirm scores save automatically and appear in call history
